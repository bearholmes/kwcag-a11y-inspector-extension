# Development Guidelines

**Note:** This file contains session-specific guidelines. For comprehensive documentation, see `CLAUDE.md` in the project root.

## Internationalization (i18n) Policy

### UI Text

- **MUST** use `chrome.i18n.getMessage()` for all user-facing UI text
- Includes: titles, labels, buttons, results headers, etc.
- Message keys defined in `_locales/{locale}/messages.json`

### Error Messages

- **MUST** be hardcoded in **English**
- Error messages are for developers/debugging purposes
- Do NOT use i18n for error messages
- Keep error messages consistent across the codebase

### Example

```typescript
// ✅ CORRECT - UI text uses i18n
const MESSAGE_KEYS = {
  CALC_TITLE: 'calcTitle',
  CALC_HEIGHT: 'calcHeight',
} as const;
header.textContent = getMessage(MESSAGE_KEYS.CALC_TITLE);

// ✅ CORRECT - Error messages in English
const ERROR_MESSAGES = {
  INVALID_INPUT: 'Invalid input value.',
  STORAGE_ERROR: 'Cannot read data from storage.',
} as const;
displayError(ERROR_MESSAGES.INVALID_INPUT);

// ❌ WRONG - Don't use i18n for error messages
displayError(getMessage('errorInvalidInput'));
```

## Default Locale

- **Default locale**: `en` (English)
- Defined in `manifest.json`: `"default_locale": "en"`
- Fallback for all missing translations

## Supported Languages

11 languages: English (en), Korean (ko), German (de), Spanish (es), French (fr), Italian (it), Japanese (ja), Portuguese (pt), Russian (ru), Chinese Simplified (zh_CN), Chinese Traditional (zh_TW)

## Code Style

### TypeScript

- Use TypeScript for all new files
- Follow strict mode rules
- Use `const` assertions for constant objects
- Prefer interfaces over type aliases for objects

### Naming Conventions

- Constants: `UPPER_SNAKE_CASE`
- Variables/Functions: `camelCase`
- Types/Interfaces: `PascalCase`
- CSS Classes: `kebab-case`

### Comments

- JSDoc for functions and public APIs
- Korean comments for complex business logic
- English for brief inline comments

## Git Workflow

- Branch naming: `claude/{feature-description}-{sessionId}`
- Commit messages in English (conventional format)
- Always commit and push before session end
- Base branch for PRs: `next`
